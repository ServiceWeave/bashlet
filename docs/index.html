Bashlet

Sandboxed bash execution using WebAssembly. Run shell commands in secure isolation with directory mounting and session management.

WASM Sandbox Secure Isolation Directory Mounting

Installation

1. Install Wasmer

Bashlet requires the Wasmer runtime for WebAssembly execution.

Shell 
curl https://get.wasmer.io -sSfL | sh

2. Install Bashlet

Quick Install (macOS/Linux) 
curl -fsSL https://bashlet.dev/install.sh | sh

Or build from source:

From Source 
git clone https://github.com/ServiceWeave/bashlet.git
cd bashlet
cargo build --release

Usage

One-Shot Execution

Run a single command in an isolated sandbox.

Shell 
# Basic command
bashlet exec "echo hello world"

# With mounted directory
bashlet exec --mount ./src:/workspace "ls /workspace"

# With environment variables
bashlet exec --env MESSAGE="Hello" "echo \$MESSAGE"

Session Management

Create persistent sandbox sessions that maintain configuration across commands.

Shell 
# Create a session
bashlet create --name my-session --mount ./src:/workspace

# Run commands in the session
bashlet run my-session "ls /workspace"
bashlet run my-session "cat /workspace/README.md"

# List active sessions
bashlet list

# Terminate when done
bashlet terminate my-session

AI Agent Mode

Run an AI agent that executes commands in the sandbox.

Shell 
# Set API key
export ANTHROPIC_API_KEY="your-api-key"

# Run agent with a task
bashlet agent "analyze the code and find potential bugs" --mount ./src:/workspace

Command Reference

Command Description
bashlet exec "cmd" One-shot command execution in sandbox
bashlet create Create a new persistent session
bashlet run SESSION "cmd" Run command in an existing session
bashlet list List all active sessions
bashlet terminate SESSION Terminate a session
bashlet agent "task" Run AI agent with sandbox access
